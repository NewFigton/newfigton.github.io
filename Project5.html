Using a Neural Net to Analyze Handwritten Digits¶

by Ciaran Lowell

Sample MNIST data

Introduction¶

One of the most common Computer Vision applications is writing code to read hand-written numbers. This problem is extremely useful in a wide range of industries, including banking, mail systems, and digitizing documents.

A popular dataset for training handwriting analysis networks and for benchmarking new classifiers is the MNIST dataset. The MNIST dataset consists of 70,000 images of handwritten digits, split into 60,000 training images and 10,000 testing images. This data is usually accessible at this link, but the website is currently inaccessible and returning a 503 error.

Instead, this project will utilize a version of this dataset hosted on Kaggle in a .csv format. This version of the dataset consists of a training .csv and a testing .csv that correspond with the original MNIST's train/test split. This link provides the code that was used to process the MNIST images and convert them into a .csv format. The data has been size-normalized and centered, which reduces the amount of pre-processing necessary for this project.

This project will create a convolutional neural network to classify hand-written arabic numerals from the MNIST dataset. The created neural net will be compared to a pre-written industry net for benchmarking.

Preliminary Data Visualization¶

The MNIST dataset consists of 70,000 images, in a 28 x 28 pixel size. The "label" column denotes which number is represented by that image. Each other column corresponds to a pixel within the image, where the cell value corresponds to a darkness value between 0 and 255. This results in a total of 784 pixels for each image.

Both the training and testing sets represent roughly equivalent numbers of images for each digit. "1" is slightly overrepresented, while "5" is slightly underrepresented.
